## 摘要
代码注释分类是自动化软件文档编写和分析中的一项关键任务。在 NLBSE'26 工具竞赛的背景下，我们提出了 LoRA-MME，这是一种利用参数高效微调（PEFT）的多模型集成架构。我们的方法通过结合四种不同的 Transformer 编码器（UniXcoder、CodeBERT、GraphCodeBERT 和 CodeBERTa）的优势，解决了 Java、Python 和 Pharo 的多标签分类挑战。通过使用低秩自适应（LoRA）独立微调这些模型，并通过学习到的加权集成策略聚合其预测结果，我们在不产生全模型微调内存开销的情况下，最大限度地提高了分类性能。我们的工具在测试集上达到了 0.7906 的加权 F1 分数和 0.6867 的宏 F1 分数。然而，集成的计算成本导致最终提交分数为 41.20%，这突显了语义准确性与推理效率之间的权衡。

## Abstract
Code comment classification is a critical task for automated software documentation and analysis. In the context of the NLBSE'26 Tool Competition, we present \textbf{LoRA-MME}, a Multi-Model Ensemble architecture utilizing Parameter-Efficient Fine-Tuning (PEFT). Our approach addresses the multi-label classification challenge across Java, Python, and Pharo by combining the strengths of four distinct transformer encoders: UniXcoder, CodeBERT, GraphCodeBERT, and CodeBERTa. By independently fine-tuning these models using Low-Rank Adaptation(LoRA) and aggregating their predictions via a learned weighted ensemble strategy, we maximize classification performance without the memory overhead of full model fine-tuning. Our tool achieved an \textbf{F1 Weighted score of 0.7906} and a \textbf{Macro F1 of 0.6867} on the test set. However, the computational cost of the ensemble resulted in a final submission score of 41.20\%, highlighting the trade-off between semantic accuracy and inference efficiency.

#### 1. 核心问题与研究动机
*   **核心问题**：如何高效且准确地对多语言（Java、Python、Pharo）源代码注释进行语义分类（如总结、用法、参数说明等）。
*   **研究动机**：代码注释包含技术术语和代码语法，通用 NLP 模型难以完全捕捉其语义。虽然预训练代码模型效果好，但全参数微调（Full Fine-Tuning）内存开销巨大。此外，不同模型对不同类别的注释捕捉能力存在差异，单一模型难以在所有类别上达到最优。

#### 2. 方法论
*   **核心思想**：结合**参数高效微调（PEFT）**与**多模型集成学习**，通过 LoRA 技术降低计算开销，并利用学习到的权重动态聚合多个代码专用编码器的预测结果。
*   **关键技术细节**：
    *   **基础模型集成**：选择了四个互补的 Transformer 编码器：**UniXcoder**（跨模态能力）、**CodeBERT**（语义对齐）、**GraphCodeBERT**（结构化数据流）和 **CodeBERTa**（轻量化）。
    *   **LoRA 微调**：在每个模型的注意力层（Query, Key, Value, Dense）注入低秩矩阵。设置秩 $r=16$，$\alpha=32$。仅需训练约 4.5% 的参数（每个模型约 5.9M 参数）。
    *   **学习型加权集成**：并非简单的概率平均，而是为每个类别 $c$ 学习一个权重向量 $W_c$，使模型能根据类别特性自动偏好表现最好的编码器（例如，GraphCodeBERT 在处理数据流相关的 Pointer 类别时权重更高）。
    *   **类别特定阈值优化**：针对多标签任务，放弃固定的 0.5 阈值，通过网格搜索为每个“语言-类别”对寻找最优 F1 阈值（范围在 0.28 到 0.85 之间）。
    *   **损失函数**：使用 **Focal Loss** 处理类别不平衡问题。

#### 3. 实验设计
*   **数据集**：NLBSE'26 竞赛数据集，包含来自 20 个开源项目的 9,361 个句子。涵盖 Java（7 类）、Python（5 类）和 Pharo（6 类）。
*   **Benchmark（基准）**：竞赛官方提供的基于 **SetFit**（Sentence-BERT + 对比学习）的基准模型。
*   **对比维度**：评估了 Precision、Recall、Macro F1 和 Weighted F1，并根据竞赛公式计算了结合准确率、推理延迟（Runtime）和计算量（GFLOPS）的综合得分。

*   **硬件**：使用了 **NVIDIA RTX 3090** GPU 进行微调。

#### 5. 实验数量与充分性
*   **实验规模**：对四种模型进行了独立微调及集成实验，并针对 18 个细分类别进行了详细的性能评估。
*   **消融实验**：专门针对“阈值优化策略”做了消融研究，证明该策略使 Macro F1 提升了 0.0355。
*   **客观性**：实验覆盖了三种截然不同的编程语言，并与官方基准进行了直接对比，结果显示在所有语言上均优于基准。

#### 6. 主要结论与发现
*   **性能优异**：LoRA-MME 在测试集上实现了 **0.7906 的加权 F1** 和 **0.6867 的宏 F1**，显著超过基准模型（整体提升 0.0359）。
*   **语言差异**：在 Python 和 Pharo 上的提升尤为明显（分别提升 0.0476 和 0.0516），证明了集成模型在处理小众或结构特殊语言时的鲁棒性。
*   **效率权衡**：尽管分类精度高，但由于集成了四个模型，导致推理速度慢、计算量大，最终竞赛综合得分仅为 41.20%，说明该方案在生产环境部署时需优化效率。

#### 7. 优点
*   **参数高效**：利用 LoRA 实现了在消费级显卡上同时微调多个大模型。
*   **动态集成**：通过学习类别权重，充分利用了不同预训练模型的结构化偏置（Inductive Bias）。
*   **精细化调优**：针对不同语言和类别独立优化阈值，有效缓解了多标签分类中的不平衡问题。

#### 8. 不足与局限
*   **计算成本高**：集成四个编码器带来了巨大的推理延迟和 GFLOPS 开销，不符合轻量化工具的竞赛导向。
*   **复杂度**：预处理阶段依赖于特定语言的启发式规则（如 Pharo 的 NEON 逻辑），可能难以泛化到其他编程语言。
*   **改进方向**：作者提到未来需要通过**知识蒸馏（Knowledge Distillation）**将集成模型的知识压缩到单个学生模型中，以平衡精度与效率。
